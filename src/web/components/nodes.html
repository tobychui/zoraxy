Nodes

Manage primary-to-worker synchronization, register new worker nodes, inspect their health, and rotate join tokens.

How Node Mode Works

The primary node stores the source-of-truth configuration. Worker nodes join with a token, fetch configuration snapshots from the primary, report heartbeat, telemetry and runtime details back, and serve only the routes assigned to them. If a worker loses access to the primary after at least one successful sync, it keeps serving the last synchronized state until the primary becomes available again.
