How to use the Route Debugger

The Route Debugger listens only on 127.0.0.1 for security. To use it:

Enable the Route Debugger above and note the port (default 5490).
In HTTP Proxy, create or edit a proxy rule and set the upstream to http://127.0.0.1:5490. You can also use a Virtual Directory rule to redirect a specific path.
Send a request through the proxy rule (from a browser, curl, or your application).
The response body contains a full dump of what Zoraxy forwarded: method, URL, all request headers (including custom headers and auth tokens), query parameters, and the request body.
When done debugging, disable the Route Debugger and restore your original upstream target.
Enable Pretty Print to get a readable HTML page when opening the proxied URL in a browser. Leave it disabled for plain-text output that works well with curl.
