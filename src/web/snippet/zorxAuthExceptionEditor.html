Zoraxy Auth - Exception Rules

Exclude specific paths or IP / CIDR ranges from Zoraxy Auth SSO authentication. Requests matching any rule below will bypass authentication entirely.

Type Pattern / CIDR Regex Trust Proxy Remove
No Exception Rule
Path prefix (e.g. /api/public/) or regex (e.g. ^/api/.*$).

IP address, CIDR block (e.g. 10.0.0.0/8), or wildcard (e.g. 192.168.1.*).

Security Warning
Enabling Trust Proxy Headers allows Zoraxy to read the client IP from HTTP headers that any client can forge. Only enable this when Zoraxy sits behind a trusted reverse proxy (e.g. Cloudflare, Nginx).

How exception rules work

  • Path prefix: Any request whose path begins with the given prefix bypasses Zoraxy Auth. The prefix must start with a slash (/).
  • Path regex: Any request whose path matches the Go regular expression bypasses Zoraxy Auth (e.g. ^/api/.*$).
  • IP / CIDR: Requests from the matching IP address, CIDR block or wildcard bypass Zoraxy Auth.
